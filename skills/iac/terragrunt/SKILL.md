# Terragrunt Infrastructure Skill

## Workflow

1. **Load context** — Read `CLAUDE.md` for project-specific Terragrunt setup
2. **Identify pattern** — Determine which Terragrunt pattern applies (catalog, live, stack)
3. **Implement** — Create/modify files following conventions
4. **Wire dependencies** — Set up `dependency {}` blocks with `mock_outputs`
5. **Validate** — Run `terragrunt validate` and verify structure

## Conventions and Rules

### Values Pattern

```hcl
inputs = {
  name           = values.name
  environment    = values.environment
  instance_type  = try(values.instance_type, "db.t3.medium")
}
```

### Reference Resolution

```hcl
inputs = {
  certificate_id = try(values.certificate_id, "") == "../acm" ?
    dependency.acm.outputs.certificate_id :
    values.certificate_id
}
```

### Module Sourcing

```hcl
# Local module
terraform {
  source = "${get_repo_root()}/infrastructure-live/modules/MODULE_NAME"
}

# Git module with version pin
terraform {
  source = "git::git@github.com:ORG/modules/mysql.git//app?ref=${values.version}"
}

# Terraform Registry module
terraform {
  source = "tfr:///hashicorp/consul/aws?version=0.1.0"
}
```

**Critical:** `//path?ref=version` — double-slash before subdirectory, `?ref=` after.

### Dependency Blocks

Every `dependency {}` MUST include `mock_outputs`:

```hcl
dependency "vpc" {
  config_path = "../vpc"
  mock_outputs = {
    vpc_id     = "mock-vpc-id"
    subnet_ids = { "public" = "mock-subnet-id" }
  }
  mock_outputs_allowed_terraform_commands = ["validate", "plan"]
}
```

### Include Pattern

```hcl
# Standard include — expose locals from root
include "root" {
  path   = "${get_repo_root()}/infrastructure-live/terragrunt.hcl"
  expose = true
}

# Access exposed values
inputs = {
  prefix = include.root.locals.prefix
  tags   = include.root.locals.tags
  env    = include.root.locals.env
}
```

### State Backend (S3-compatible)

```hcl
# Generated by root terragrunt.hcl
generate "backend" {
  path      = "backend.tf"
  if_exists = "overwrite_terragrunt"
  contents  = <<EOF
terraform {
  backend "s3" {
    bucket         = "terragrunt-state-ACCOUNT-PREFIX"
    key            = "${path_relative_to_include()}/tf.tfstate"
    region         = "REGION"
    encrypt        = true
    dynamodb_table = "terragrunt-locks"
  }
}
EOF
}
```

### Before Hooks

```hcl
terraform {
  before_hook "validate" {
    commands = ["plan", "apply"]
    execute  = ["terraform", "validate"]
  }
}
```

## Practical Examples

### Creating a New Component

```hcl
# infrastructure-live/accounts/ACCOUNT_NAME/ENV_NAME/REGION/components/NEW_COMPONENT/terragrunt.hcl

include "root" {
  path   = "${get_repo_root()}/infrastructure-live/terragrunt.hcl"
  expose = true
}

terraform {
  source = "${get_repo_root()}/infrastructure-live/modules/MODULE_NAME"
}

dependency "vpc" {
  config_path = "../vpc"
  mock_outputs = {
    vpc_id     = "mock-vpc-id"
    subnet_ids = ["mock-subnet-id"]
  }
  mock_outputs_allowed_terraform_commands = ["validate", "plan"]
}

inputs = {
  vpc_id       = dependency.vpc.outputs.vpc_id
  display_name = "resource-${include.root.locals.env}"

  tags = merge(include.root.locals.tags, { ManagedBy = "Terraform" })
}
```

### Using generate for Provider Pins

```hcl
# Required when using registry/github modules that need specific provider versions
generate "versions_override" {
  path      = "versions_override.tf"
  if_exists = "overwrite_terragrunt"
  contents  = <<EOF
terraform {
  required_providers {
    # Replace with your cloud provider
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
EOF
}
```

### Heredoc in Ternary (Common Pitfall)

```hcl
# WRONG — breaks parsing
inputs = {
  config = var.enabled ? <<EOF
    content
  EOF
  : ""
}

# CORRECT — wrap in parentheses
inputs = {
  config = var.enabled ? (<<EOF
    content
  EOF
  ) : ""
}
```

### run-all with Dependency Ordering

```bash
# Plan all components respecting dependency order
terragrunt run-all plan

# Apply specific component and its dependencies
terragrunt run-all apply --terragrunt-include-dir "components/kubernetes"

# Destroy in reverse dependency order
terragrunt run-all destroy
```

## DO NOT

- **DO NOT** use `include "envcommon"` for module source — always hardcode `terraform { source }`
- **DO NOT** omit `mock_outputs` from `dependency {}` blocks — breaks CI/CD `plan`/`validate`
- **DO NOT** put `?ref=` before `//path` in git sources — causes refspec errors
- **DO NOT** use floating version refs (branches) in production — always pin tags
- **DO NOT** use `run-all` without understanding the dependency graph first
- **DO NOT** store secrets in `terragrunt.hcl` — use `env_local.hcl` (gitignored)
- **DO NOT** use `--terragrunt-parallelism` > 4 without `TF_PLUGIN_CACHE_DIR` set
- **DO NOT** create `versions.tf` — use `required_providers` in `main.tf` or `generate` blocks
- **DO NOT** skip `before_hook "validate"` — catches errors before plan/apply

## Validation Commands

```bash
# Validate a single component
cd infrastructure-live/accounts/ACCOUNT_NAME/ENV_NAME/REGION/components/COMPONENT
terragrunt validate

# Plan (dry-run)
terragrunt plan

# First-time deploy (bootstraps backend bucket)
terragrunt plan --backend-bootstrap

# Format check
terraform fmt -check -recursive infrastructure-live/modules/

# Validate all components (from infrastructure-live/)
terragrunt run-all validate

# Show dependency graph
terragrunt graph-dependencies
```

## Common Pitfalls

| Pitfall | Symptom | Fix |
|---------|---------|-----|
| Git refspec error | `?ref=` before `//path` | Move `?ref=` after `//path` |
| Heredoc in ternary | Parse error | Wrap heredoc in parentheses |
| Missing mock_outputs | Plan fails on new stack | Add `mock_outputs` + `mock_outputs_allowed_terraform_commands` |
| Floating version refs | Unexpected changes | Pin git tags in production |
| Provider race condition | `run-all init` failures | Set `TF_PLUGIN_CACHE_DIR` |
| Stale cache | Unexpected source | Run `make clean` or delete `.terragrunt-cache` |
